Real-Time Implementation

Scope: Bradykinesia · Dyskinesia · Tremor
Overview

The consultant's implementation used an offline, batch-processing pipeline. This document describes the key differences introduced in the integrated re-implementation, which operates under real-time constraints with limited resources. Where implementation choices differ, the rationale is provided, as well as accuracy implications observed against the validated Python ground truth.

Accuracy Against Python Ground Truth

Three scores are derived from the resampled accG signal. The following results are based on ground truth validation tests.

Dyskinesia sensitivity note: The dyskinesia score is particularly sensitive to resampling inaccuracies. The very small resampling inaccuracies (Change 1) are amplified in the order of 105 by this metric. Thus higher scores come with higher errors, sometimes hitting the error threshold.
Implementation Changes
1
Resampling Strategy & Filter State
Highest Impact
Consultant Implementation
  • Processed accG data in 2-minute blocks with 10-second padding on each side; only the central 2 minutes were used for score computation
  • Stateless filter, warmed up on the 10-second padding and reset at the start of every new 2-minute block
  • First 2-minute block was left-padded with zeros
Integrated Implementation
  • Processes 1 second of data at a time with 0.5-second padding on each side, requiring 0.5 seconds of future data
  • Stateful filter, state preserved across successive 1-second windows
  • First second of data is left-padded with a mirrored version of itself
⚠ Resampling each second block instead of every 2 minutes introduces a measurable performance overhead. Additionally, resampling results are not identical to the consultant implementation, even though the numerical differences are very small.
2
Data Type Standardization
Precision Trade-off
Consultant Implementation
  • Mixed usage of double, float, and int throughout
Integrated Implementation
  • Standardized to float32_t and stdint types throughout
  • double / float64 selectively replaced with float32_t
⚠ Numerically fragile operations (filter pole/zero computation, complex arithmetic) retain full precision. Float32 was used instead in sections where precision was not crucial for the accuracy of the results.
3
Tremor Detection at Block Boundaries
Correctness Fix
Consultant Implementation
  • Incorrect number of 5-second subwindows was computed, which caused some tremor streaks to not be detected
  • Required a custom heuristic to handle tremors crossing 2-minute block boundaries
Integrated Implementation
  • Number of considered windows is correct
  • No boundary heuristic required; detection is continuous by design
4
Tremor Score — 75th Percentile of Array
Correctness Fix
Consultant Implementation
  • Missing additional 0 to the array of 1-second power-spectrum peaks before computing the 75th percentile
Integrated Implementation
  • A zero is prepended to the score array before computing the 75th percentile
  • Ensures that a single-element array yields a percentile of 75% of that value, matching Python ground truth behavior
